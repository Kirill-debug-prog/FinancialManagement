Веб-приложения для учета персональных финансов
1. Общие положения
1.1. Наименование проекта
Рабочее название: "FinanceTracker"
Полное название: Веб-приложение для персональных финансов с аналитикой и визуализацией расходов.
1.2. Основания для разработки
Проект разрабатывается по инициативе заказчика с целью создания современного инструмента для управления личными финансами.
1.3. Цель проекта
Создать удобное, интуитивно понятное и безопасное веб-приложение, которое поможет пользователям:
Эффективно отслеживать свои доходы и расходы.
Анализировать финансовые привычки.
Планировать бюджет и ставить финансовые цели.
Принимать взвешенные решения для улучшения своего финансового положения.
1.4. Целевая аудитория
Молодые специалисты и студенты: Люди, которые начинают вести самостоятельный бюджет и хотят выработать правильные финансовые привычки.
Семьи: Пользователи, желающие контролировать общий семейный бюджет, оптимизировать расходы и делать совместные накопления.
Фрилансеры и самозанятые: Люди, которым необходимо вести учет личных финансов, отделяя их от рабочих.
Все, кто хочет повысить свою финансовую грамотность: Пользователи, которые ищут простой инструмент для наглядного представления своих финансовых потоков.
1.5. Основные задачи, решаемые приложением
Учет: Предоставить простой и быстрый способ добавления всех финансовых операций (доходов, расходов, переводов).
Категоризация: Дать возможность гибко настраивать и присваивать категории расходам для последующего анализа.
Визуализация: Наглядно представлять финансовые данные в виде понятных графиков и диаграмм.
Планирование: Обеспечить функционал для создания и контроля месячных или недельных бюджетов по различным категориям.
Доступность: Гарантировать полноценную и удобную работу с приложением как на настольных компьютерах, так и на мобильных устройствах через веб-браузер (адаптивный дизайн).
2. Функциональные требования
2.1. Модуль "Регистрация и Аутентификация"
Этот модуль описывает процесс создания персонального аккаунта пользователя, входа в систему и управления доступом.
2.1.1. Приветственный экран (Стартовый экран для новых пользователей)
Описание: Первый экран, который видит пользователь, не выполнивший вход в систему. Вместо локального онбординга он предлагает создать аккаунт или войти в существующий.
Логика: Система проверяет, авторизован ли пользователь. Если нет — показывает этот экран.
Предлагаемый интерфейс (на основе вашей идеи):
Заголовок: "Добро пожаловать в FinanceTracker!".
Краткое описание преимуществ: "Начните контролировать свои финансы. Отслеживайте расходы, планируйте бюджеты и достигайте финансовых целей".
Кнопка "Создать аккаунт" (основное действие): Переводит пользователя на экран регистрации.
Ссылка/Кнопка "У меня уже есть аккаунт" (второстепенное действие): Переводит пользователя на экран входа (авторизации).
2.1.2. Экран Регистрации
Описание: Форма для создания нового онлайн-аккаунта. Это заменяет "Карточку пользователя" из локальной версии.
Визуальное представление: Можно представить как классическую форму регистрации.
Составные элементы (поля ввода):
Поле "Имя":
Тип: Текстовое поле.
Валидация: Обязательно для заполнения.
Поле "Email":
Тип: Поле для ввода электронной почты.
Валидация: Обязательно для заполнения. Проверка на корректность формата (наличие @ и домена). Система должна проверять на сервере уникальность данного Email (нельзя зарегистрировать два аккаунта на одну почту).
Поле "Пароль":
Тип: Поле для ввода пароля (символы маскируются).
Валидация: Обязательно для заполнения. Минимальная длина 8 символов, должен содержать хотя бы одну цифру и одну букву.
Поле "Подтвердите пароль":
Тип: Поле для ввода пароля (символы маскируются).
Валидация: Обязательно для заполнения. Значение должно совпадать с полем "Пароль".
Чекбокс "Я согласен с Условиями использования и Политикой конфиденциальности":
Текст "Условия использования" и "Политика конфиденциальности" должен быть представлен в виде ссылок на соответствующие документы.
Валидация: Чекбокс должен быть отмечен для продолжения регистрации.
Функциональные элементы:
Кнопка "Зарегистрироваться":
Выполняет клиентскую валидацию полей. При ошибках — выводит подсказки.
Если все верно, отправляет данные на сервер.
Сервер выполняет свою валидацию (включая проверку уникальности Email).
При успехе:
Создает новый аккаунт пользователя в базе данных.
Отправляет на указанный Email письмо для подтверждения регистрации (со ссылкой).
Перенаправляет пользователя на экран "Подтвердите ваш Email".


Ссылка "Войти": Перенаправляет на экран авторизации.
2.1.3. Экран "Подтвердите ваш Email"
Описание: Информационный экран, который сообщает пользователю о необходимости проверить почту и перейти по ссылке в письме для активации аккаунта.
Составные элементы:
Иконка письма.
Текст: "Мы отправили письмо на [Email пользователя]. Пожалуйста, перейдите по ссылке в письме, чтобы завершить регистрацию."
Кнопка/ссылка "Отправить письмо повторно" (становится активной через 60 секунд после первой отправки).
2.1.4. Экран Авторизации (Входа)
Описание: Форма для входа в существующий аккаунт.
Поля ввода: "Email", "Пароль".
Функционал:
Кнопка "Войти".
Ссылка "Забыли пароль?" (запускает процесс восстановления доступа).
Ссылка "Зарегистрироваться" (переводит на экран регистрации).
2.1.6. Создание первого профиля после регистрации (ВАЖНОЕ ДОПОЛНЕНИЕ)
Логика: Сразу после того,как пользователь впервые успешно зарегистрировался и вошел в свой аккаунт, система должна предложить ему создать свой первый профиль.
Интерфейс: Открывается упрощенная форма, аналогичная "Карточке пользователя", но без полей для пароля, так как безопасность обеспечивается на уровне аккаунта.
Поле "Название профиля": (например, "Мои финансы").
Выбор основной валюты для профиля.
Кнопка "Создать и начать работу".
Результат: После создания первого профиля пользователь попадает на главный экран приложения, и все дальнейшие действия (создание счетов, транзакций) происходят уже внутри этого профиля.
2.2. Шаг 2: Настройка валют
Описание: Экран, на котором пользователь выбирает валюты, которые он будет использовать для ведения счетов и учета транзакций.
Составные элементы:
Индикатор прогресса: "Шаг 2 из 3".
Заголовок: "Добавить валюту".
Информационный блок: "Сейчас активны следующие валюты:". Изначально пуст. После добавления первой валюты здесь будет отображаться ее название (например, "Российский рубль").
Функциональные элементы:
Кнопка "ДОБАВИТЬ ВАЛЮТУ": Открывает экран "Валюта" для настройки и добавления.
Кнопка "ДАЛЕЕ": Неактивна до тех пор, пока не добавлена хотя бы одна валюта. После добавления становится активной и ведет на "Шаг 3: Настройка счетов".
Кнопка "НАЗАД": Возвращает на предыдущий шаг.
2.2.1. Форма добавления/редактирования валюты
Описание: Детальная форма для настройки параметров валюты.
Логика работы:
Система должна иметь предустановленный справочник мировых валют (RUB, USD, EUR, KZT, BYN и т.д.).
При выборе валюты из списка "Выбор валюты из списка" все остальные поля ("Код", "Название", "Сокращение") должны заполняться автоматически значениями по умолчанию.
Пользователь может вручную отредактировать эти значения.
Поля и элементы:
"Выбор валюты из списка": Выпадающий список с предустановленными мировыми валютами.
"Порядковый номер в списке": Числовое поле, определяет порядок сортировки валют в других списках.
"Название полностью": Текстовое поле, например, "Российский рубль".
"Код": Текстовое поле для международного кода, например, "RUB".
"Название сокращенно": Текстовое поле для символа валюты, например, "₽" или "р.".
"Количество знаков после запятой в суммах": Выпадающий список/числовое поле (обычно 2).
"Количество знаков после запятой в курсах валют": Выпадающий список/числовое поле (обычно 4).
Кнопка "ОК": Сохраняет валюту в профиле пользователя и возвращает на предыдущий экран.
Кнопка "ОТМЕНА": Закрывает форму без сохранения.
2.3. Шаг 3: Настройка счетов
Описание: Экран, где пользователь создает свои первые финансовые счета (например, кошелек, банковская карта).
Составные элементы:
Индикатор прогресса: "Шаг 3 из 3".
Заголовок: "Настройка счетов".
Список добавленных счетов (изначально пуст).
Функциональные элементы:
Кнопка "ДОБАВИТЬ СЧЕТ": Открывает экран "Счет" для создания нового счета.
Кнопка "ДАЛЕЕ": Неактивна, пока не добавлен хотя бы один счет. После добавления ведет на финальный экран "Все готово!".
Кнопка "НАЗАД": Возвращает на шаг настройки валют.
2.3.1. Форма добавления/редактирования счета
Описание: Форма для ввода данных о финансовом счете.
Поля и элементы:
"Название счета": Текстовое поле. Обязательно для заполнения (например, "Карта Сбербанк", "Наличные").
"Картинка": Выбор иконки для счета из предустановленной библиотеки (иконки кошелька, банков, карт и т.д.).
"Номер в списке": Числовое поле для сортировки.
"Начальный баланс":
Выбор валюты: Выпадающий список, содержащий валюты, добавленные пользователем на предыдущем шаге.
"Сумма": Числовое поле для ввода текущего остатка на счете.
"Дата": Поле выбора даты, на которую актуален этот баланс. По умолчанию — текущая дата.
"Примечание": Необязательное текстовое поле для дополнительной информации.
Кнопка "ОК": Сохраняет счет. При сохранении система должна автоматически создать первую "системную" транзакцию типа "Начальный баланс" для этого счета.
Кнопка "ОТМЕНА": Закрывает форму без сохранения.
2.4. Завершение настройки
Описание: Финальный экран онбординга, информирующий пользователя об успешном завершении первоначальной настройки.
Функциональные элементы:
Кнопка "ГОТОВО": Завершает процесс настройки и перенаправляет пользователя на главный экран приложения.
Кнопка "НАЗАД": Возвращает на шаг настройки счетов.
3. Основной интерфейс приложения
3.1. Главный экран (Дашборд)
Описание: Центральный экран приложения, который служит точкой входа ко всем основным функциям. Представляет собой сетку иконок-модулей.
Структура экрана:
3.1.1. Верхняя панель (Header):
Иконка синхронизации (облако со стрелками).
Иконка настроек (шестеренка).
Иконка "три точки" (дополнительное меню).
3.1.2. Сетка модулей: Каждая иконка является кнопкой для перехода в соответствующий раздел.
"Счета": Переход к списку всех счетов, их балансов и операций по ним.
"Пользователи": В MVP ведет в настройки профиля текущего пользователя. В будущем - управление семейным доступом.
"Валюты": Переход в раздел управления валютами (добавление, редактирование, удаление).
"Отчёты": Переход в модуль аналитики и визуализации данных.
"Расходы": Быстрый переход к форме добавления транзакции с типом "Расход".
"Доходы": Быстрый переход к форме добавления транзакции с типом "Доход".
"Получено в кредит" / "Выдано в долг": Переход в модуль учета долгов.
"Планирование расходов" / "Планирование доходов": Вероятно, модуль для постановки финансовых целей. Требует уточнения.
"Бюджет расходов" / "Бюджет доходов": Переход в модуль бюджетирования (установка лимитов).
"Категории расходов" / "Категории доходов": Раздел для управления категориями транзакций.
"Единицы измерения": Нестандартный модуль. Требует уточнения. Возможно, для учета нефинансовых активов (например, литры бензина).
"Кредиторы и должники": Справочник контактов для модуля учета долгов.
3.1.3. Плавающая кнопка действия (FAB):
Круглая синяя кнопка с + в правом нижнем углу.
Назначение: Основное действие на экране. При нажатии должна открываться форма быстрого добавления транзакции (расход, доход или перевод между счетами).
3.2. Модуль "Счета"
3.2.1. Главный экран модуля "Счета"
Назначение: Экран предназначен для отображения списка всех созданных пользователем финансовых счетов, их текущих балансов и для навигации к операциям по этим счетам.
Составные элементы экрана:
A. Верхняя панель (Header):
Иконка "Бургер-меню" (≡): При нажатии открывает главную навигационную панель/меню приложения (возврат на главный экран с иконками, переход в настройки и т.д.).
Заголовок: "Счета".
Иконка "Три точки" (⋮): При нажатии открывает контекстное меню с опциями для данного экрана, такими как:
"Добавить счет" (альтернативный способ вызова формы создания счета).
"Настроить сортировку".
"Показать архивные счета".
B. Панель вкладок:
Позволяет пользователю переключаться между различными представлениями данных о счетах.
Вкладка "КРАТКО": Выбрана по умолчанию. Отображает сводный список счетов. Описана подробно ниже.
Вкладка "ПОДРОБНО": [Заготовка] Функционал данной вкладки будет определен на следующих этапах. Предполагается, что здесь будет отображаться список счетов с дополнительной информацией (например, с графиком изменения баланса за месяц).
Вкладка "ПЕРЕНОСЫ СО СЧЁТА НА СЧЁТ": [Заготовка] Функционал данной вкладки будет определен на следующих этапах. Предполагается, что здесь будет отображаться история всех переводов между счетами пользователя.
C. Список счетов (содержимое вкладки "КРАТКО"):
Описание: Основная область экрана, где выводится перечень всех активных счетов пользователя.
Структура элемента списка: Каждая строка в списке представляет один счет и содержит:
Иконку счета (выбранную при создании).
Название счета (например, "Карта Tinkoff").
Текущий баланс счета с сокращенным названием валюты (например, "15 240,50 р").
Поведение:
При нажатии (клике/тапе) на строку счета, пользователь переходит на Экран редактирования счета (см. п. 3.2.2).
[На будущее] Длинное нажатие (long tap) на мобильных устройствах может открывать контекстное меню для быстрого действия (например, "Создать расход по этому счету", "Архивировать").
D. Плавающая кнопка действия (FAB):
Описание: Круглая синяя кнопка с иконкой + в правом нижнем углу экрана.
Действие: При нажатии открывает экран "Создание новой транзакции".
E. Нижняя панель (Footer) с итоговой информацией:
Описание: Отображает агрегированные данные по всем счетам, видимым в списке, за выбранный период (по умолчанию — текущий месяц).
Поля:
Расход: Сумма всех расходных операций за период.
Доход: Сумма всех доходных операций за период.
Остаток: Суммарный баланс всех счетов на текущий момент.
3.2.2. Экран редактирования счета
Назначение: Предоставить пользователю возможность изменять параметры существующего счета.
Способ вызова: Открывается по нажатию на счет из списка на экране "Счета".
Структура и поля:
Форма должна быть идентична форме создания счета, но предзаполнена текущими данными выбранного счета.
Редактируемые поля:
"Название счета".
"Картинка" (иконка).
"Номер в списке" (для сортировки).
"Примечание".
Нередактируемые поля:
"Начальный баланс" и "Валюта": Эти поля не могут быть изменены после создания счета и проведения по нему транзакций во избежание нарушения целостности данных. Для изменения текущего баланса пользователь должен создать транзакцию "Корректировка" или "Доход/Расход".
Функциональные элементы:
Кнопка "ОК" / "СОХРАНИТЬ": Сохраняет внесенные изменения и возвращает пользователя на экран со списком счетов.
Кнопка "ОТМЕНА": Отменяет все изменения и возвращает пользователя на предыдущий экран.
Кнопка/ссылка "УДАЛИТЬ СЧЕТ":
При нажатии система должна запросить подтверждение действия ("Вы уверены, что хотите удалить счет [Название счета]?").
Бизнес-правило: Удаление счета возможно только в том случае, если по нему не было проведено ни одной транзакции (за исключением системной транзакции "Начальный баланс").
Если транзакции есть, вместо удаления должна быть доступна функция "АРХИВИРОВАТЬ СЧЕТ". Архивированный счет скрывается из всех основных списков и отчетов, но его данные сохраняются в системе.
3.3. Модуль "Профили (Управление)"
3.3.1. Экран управления профилями
Назначение: Предоставить пользователю интерфейс для создания новых, редактирования существующих и удаления ненужных профилей внутри его аккаунта.
Способ вызова:
Из главного меню (дашборда с иконками) по нажатию на иконку "Пользователи" (которую в нашей системе следует переименовать в "Профили").
Из выпадающего списка переключения профилей в шапке приложения -> пункт "Управление профилями".
Визуальное представление: Экран представляет собой список всех созданных профилей с элементами управления. За основу можно взять компоновку с присланного скриншота, адаптировав ее под задачу управления, а не входа.
Составные элементы экрана:
A. Верхняя панель (Header):
Кнопка "Назад": Возвращает пользователя на предыдущий экран (например, на главный дашборд).
Заголовок: "Управление профилями".
B. Список профилей:
Описание: Основная область экрана, где выводится перечень всех профилей, созданных в рамках текущего аккаунта.
Структура элемента списка: Каждая строка в списке представляет один профиль и содержит:
Название профиля (например, "Личные финансы").
Индикатор активного профиля (например, пометка "(активный)" или специальная иконка/цвет фона).
Элементы управления:
Кнопка/Иконка "Редактировать" (карандаш): При нажатии открывает форму редактирования профиля.
Кнопка/Иконка "Удалить" (корзина): При нажатии запускает процесс удаления профиля.
C. Функциональные элементы:
Кнопка "ДОБАВИТЬ ПРОФИЛЬ":
Расположена внизу экрана или в виде плавающей кнопки действия (FAB) с +.
При нажатии открывает форму создания нового профиля.
3.3.2. Форма создания/редактирования профиля
Назначение: Модальное окно или отдельный экран для ввода/изменения данных профиля.
Поля и элементы:
Заголовок: "Новый профиль" или "Редактирование профиля".
Поле "Название профиля":
Тип: Текстовое поле.
Валидация:
Обязательно для заполнения.
Название должно быть уникальным в рамках одного аккаунта.
Максимальная длина - 50 символов.
Поле "Основная валюта":
Логика: Это поле доступно только при создании нового профиля. После создания валюту профиля менять нельзя, чтобы не нарушить финансовые данные.
Тип: Выпадающий список, содержащий предустановленный перечень основных валют (RUB, USD, EUR и т.д.).
Управляющие кнопки:
Кнопка "СОХРАНИТЬ" / "ОК":
Проверяет валидность данных.
Сохраняет (создает или обновляет) профиль в базе данных.
Закрывает форму и обновляет список на экране управления профилями.
Кнопка "ОТМЕНА": Закрывает форму без сохранения изменений.
3.3.3. Процесс удаления профиля
Назначение: Обеспечить безопасное удаление профиля со всеми связанными данными.
Логика:
Пользователь нажимает иконку "Удалить" у выбранного профиля.
Система отображает модальное окно с предупреждением: "Вы уверены, что хотите удалить профиль '[Название профиля]'? Все связанные с ним счета, транзакции, бюджеты и отчеты будут удалены безвозвратно. Это действие нельзя отменить."
Для подтверждения удаления пользователь должен вручную ввести название удаляемого профиля в специальное текстовое поле.
После корректного ввода названия кнопка "УДАЛИТЬ" в модальном окне становится активной.
При нажатии на кнопку "УДАЛИТЬ" система выполняет полное удаление профиля и всех его данных из базы данных.
Бизнес-правило: Нельзя удалить последний оставшийся в аккаунте профиль. Система должна выводить сообщение: "Невозможно удалить единственный профиль. Вы можете переименовать его или создать новый."
3.5. Модуль "Отчёты"
3.5.1. Главный экран модуля "Отчёты"
Назначение: Экран служит для отображения списка ранее сформированных и сохраненных отчетов, а также для запуска процесса создания новых отчетов.
Способ вызова: Нажатие на иконку "Отчёты" на главном экране (дашборде).
Составные элементы экрана:
A. Верхняя панель (Header):
Иконка "Бургер-меню" (≡): Открывает главную навигационную панель.
Заголовок: "Отчёты".
Иконка "Список отчетов" (📄): Ключевой элемент. При нажатии открывает меню или выпадающий список с перечнем типов доступных для формирования отчетов (см. п. 3.5.3).
Иконка "Три точки" (⋮): Открывает контекстное меню для управления списком сохраненных отчетов (например, "Удалить все отчеты").
B. Область контента:
Состояние "Список пуст" (Empty State):
Отображается, если пользователь еще не сформировал и не сохранил ни одного отчета.
Содержит текст: "Список пуст. Для создания отчета нажмите Меню [имеется в виду иконка 📄] и выберите нужный отчет".
Состояние "Список с отчетами" (Populated State):
Когда отчеты сформированы, они отображаются здесь в виде списка.
Структура элемента списка: Каждая строка — это сохраненный отчет, содержащий:
Название отчета (например, "Расходы по категориям за Май 2024").
Дата формирования отчета.
Тип отчета (например, "Круговая диаграмма").
Поведение:
Нажатие: Открывает экран просмотра ранее сформированного отчета.
Длительное нажатие: Открывает контекстное меню для конкретного отчета ("Удалить", "Экспортировать", "Обновить").
3.5.2. Процесс создания нового отчёта (User Flow)
Пользователь нажимает на иконку "Список отчетов" (📄) в шапке.
Появляется меню с выбором типа отчета (например, "Доходы и расходы", "Структура расходов").
Пользователь выбирает нужный тип.
Открывается экран "Параметры отчета", где пользователь настраивает детали (например, период, счета).
Пользователь нажимает кнопку "Сформировать".
Открывается экран "Просмотр отчета" с визуализацией и табличными данными.
На экране просмотра есть кнопка "Сохранить", при нажатии на которую отчет появляется в списке на главном экране модуля "Отчёты".
3.5.3. Перечень типов отчётов (для MVP)
Меню, вызываемое по иконке 📄, должно содержать следующие предустановленные типы отчетов:
1. Структура расходов: Круговая или столбчатая диаграмма, показывающая долю каждой категории в общей сумме расходов за период.
2. Структура доходов: Аналогично расходам, но для категорий доходов.
3. Динамика доходов и расходов: Линейный график, где двумя линиями показаны суммы доходов и расходов по дням/неделям/месяцам.
4. Движение средств: Детальная табличная выписка всех операций (доходы, расходы, переводы) за выбранный период.
5. Отчет по счетам: Таблица с начальным балансом, суммой поступлений, суммой списаний и конечным балансом для каждого счета за период.
3.5.4. Экран "Параметры отчета"
Назначение: Позволяет пользователю гибко настроить данные для формирования отчета.
Обязательные поля для всех типов отчетов:
Период:
Выбор из предустановленных диапазонов ("Текущий месяц", "Прошлый месяц", "Текущий год").
Возможность выбрать произвольный диапазон дат ("С... По...").
Счета:
Возможность выбрать все счета или отметить галочками конкретные счета для включения в отчет.
Валюта отчета:
Выпадающий список, позволяющий выбрать, в какой валюте отображать итоговые суммы (по умолчанию — основная валюта профиля). Система должна производить конвертацию по заданным курсам.
Дополнительные поля (зависят от типа отчета):
Для отчета по категориям — возможность выбрать конкретные категории.
3.5.5. Экран "Просмотр отчета"
Назначение: Отображение сформированного отчета.
Составные элементы:
Заголовок: Название отчета и период (например, "Структура расходов (01.05.24 - 31.05.24)").
Область визуализации: График или диаграмма. Элементы должны быть интерактивными (при наведении мыши/тапе показывать точные суммы).
Область данных: Таблица с детализацией цифр, которые легли в основу визуализации.
Кнопки действий: "Сохранить отчет", "Экспорт в CSV/PDF", "Поделиться".
3.6. Модуль "Расходы"
3.6.1. Экран списка расходов
Назначение: Отображение списка всех расходных операций за выбранный период. Предоставление инструментов для навигации, фильтрации и добавления новых расходов.
Способ вызова: Нажатие на иконку "Расходы" на главном экране (дашборде).
Составные элементы экрана:
A. Верхняя панель:
Иконка "Бургер-меню": Открывает главную навигационную панель.
Заголовок: "Расходы".
Иконка "Поиск": При нажатии открывает текстовое поле для поиска по полю "Примечание" в транзакциях.
Иконка "Фильтр": При нажатии открывает экран "Фильтр". Если фильтры применены, иконка должна изменять цвет/вид, сигнализируя об активной фильтрации.
B. Навигатор по периодам:
Описание: Интерактивный элемент для быстрой смены отображаемого периода.
Элементы:
Стрелки "<" и ">": Переключают на предыдущий/следующий период (например, на предыдущий/следующий месяц).
Название периода ("Октябрь 2025"): При нажатии открывает модальное окно/меню для выбора типа периода:
День
Неделя
Месяц (выбран по умолчанию)
Год
Всё время
Другой период (открывает календарь для выбора произвольного диапазона дат).
C. Список расходов:
Состояние "Список пуст": Отображается, если за выбранный период нет расходных операций. Содержит текст: "Список пуст. Для добавления записи нажмите на кнопку '+'".
Состояние со списком: Когда транзакции есть, они отображаются в виде списка, где каждый элемент содержит: дату, иконку и название категории, примечание (если есть) и сумму расхода.
Поведение: Нажатие на запись в списке открывает ее для редактирования в форме "Карточка расходов".
D. Плавающая кнопка действия (FAB) и ее опции:
Описание: Круглая синяя кнопка, которая при нажатии раскрывается, предоставляя несколько вариантов действий.
Изначальное состояние: Кнопка с иконкой +.
При нажатии на +: Кнопка превращается в X (закрыть) и рядом появляются дочерние кнопки:
"Добавить расход": Основное действие. Открывает экран "Карточка расходов" для создания новой записи.
"QR-код": [Функционал для будущих версий] При нажатии активирует камеру устройства для сканирования QR-кода с кассового чека. Система должна попытаться автоматически распознать и заполнить поля в карточке расхода (сумма, дата, товары).
E. Итоговая панель:
Отображает общую сумму всех расходов, отображенных в списке (с учетом выбранного периода и фильтров).
3.6.2. Форма добавления/редактирования расхода ("Карточка расходов")
Назначение: Детальная форма для ввода всех данных о расходной операции.
Поля и элементы:
"Дата": Поле для выбора даты операции. Имеет кнопки быстрого выбора "Сегодня", "Вчера". При нажатии на само поле открывается календарь. Обязательно для заполнения.
"Списать со счёта": Выпадающий список, содержащий все активные счета текущего профиля. Обязательно для заполнения.
"Категория": Выпадающий список категорий расходов. Обязательно для заполнения.
"Подкатегория": Выпадающий список, содержимое которого зависит от выбранной "Категории". Если у категории нет подкатегорий, поле неактивно или скрыто.
"Сумма": Числовое поле для ввода основной суммы расхода. Обязательно для заполнения.
"Валюта": Выпадающий список валют, добавленных в профиле. По умолчанию подставляется валюта выбранного счета.
"Скидка, %": Необязательное числовое поле. Итоговая сумма транзакции будет рассчитана как Сумма * (1 - Скидка / 100).
"Количество": Числовое поле, по умолчанию "1". Позволяет указывать количество товара/услуги. Итоговая сумма транзакции будет (Сумма * Количество) * (1 - Скидка / 100).
"Единица изм.": Выпадающий список, связанный с отдельным справочником "Единицы измерения" (например, шт., кг, л).
"Примечание": Необязательное текстовое поле для любых комментариев.
Управляющие кнопки:
"ОК": Выполняет валидацию полей. При успехе сохраняет транзакцию, списывает сумму с баланса выбранного счета и возвращает на экран списка расходов.
"ОТМЕНА": Закрывает форму без сохранения данных.
"ЕЩЕ": Сохраняет текущую транзакцию (аналогично "ОК"), но не закрывает форму, а очищает ее для ввода следующей. Для удобства поля "Дата" и "Списать со счёта" могут оставаться предзаполненными с предыдущей записи.
3.6.3. Экран "Фильтр"
Назначение: Предоставить пользователю возможность отобрать и отобразить в списке только те расходы, которые соответствуют заданным критериям.
Составные элементы:
Кнопка "СБРОСИТЬ ФИЛЬТР": Снимает все галочки и очищает все поля в фильтре.
Список критериев фильтрации:
"Счета": Список всех счетов с чекбоксами для выбора.
"Категории": Иерархический (древовидный) список категорий и подкатегорий. Пользователь может выбрать как всю категорию целиком, так и отдельные подкатегории.
Логика работы фильтра:
Внутри одной группы (например, "Счета") применяется логика "ИЛИ" (показывать расходы со Счета 1 ИЛИ Счета 2).
Между разными группами ("Счета" и "Категории") применяется логика "И" (показывать расходы со Счета 1 И относящиеся к Категории "Продукты").
Управляющие кнопки:
"ОК": Применяет выбранные настройки фильтра, закрывает экран и обновляет список расходов в соответствии с новыми критериями.
"ОТМЕНА": Закрывает экран фильтра без применения изменений.
3.7. Модуль "Доходы"
3.7.1. Экран списка доходов
Назначение: Отображение списка всех доходных операций за выбранный период. Предоставление инструментов для навигации, фильтрации и добавления новых доходов.
Способ вызова: Нажатие на иконку "Доходы" на главном экране (дашборде).
Визуальное представление: Аналогично экрану "Расходы", с заменой заголовков и терминологии.
Составные элементы экрана:
A. Верхняя панель (Header):
Иконка "Бургер-меню": Открывает главную навигационную панель.
Заголовок: "Доходы".
Иконка "Поиск" : Открывает текстовое поле для поиска по полю "Примечание" в транзакциях.
Иконка "Фильтр": Открывает экран "Фильтр". Если фильтры применены, иконка меняет свой вид.
B. Навигатор по периодам:
Функционал полностью идентичен модулю "Расходы": переключение стрелками и выбор типа периода (День, Неделя, Месяц и т.д.).
C. Список доходов:
Состояние "Список пуст": Отображается, если за выбранный период нет доходных операций.
Состояние со списком: Отображает список транзакций дохода, где каждый элемент содержит: дату, категорию, примечание и сумму дохода.
Поведение: Нажатие на запись в списке открывает ее для редактирования в форме "Карточка доходов".
D. Плавающая кнопка действия (FAB):
Описание: Круглая синяя кнопка, которая при нажатии раскрывается.
Действие при раскрытии: Появляется кнопка "Добавить доход", которая открывает экран "Карточка доходов" .
E. Итоговая панель:
Отображает общую сумму всех доходов, отображенных в списке (с учетом периода и фильтров).
3.7.2. Форма добавления/редактирования дохода ("Карточка доходов")
Назначение: Детальная форма для ввода всех данных о доходной операции.
Визуальное представление: Аналогично "Карточке расходов", с соответствующими изменениями в полях.
Поля и элементы:
"Дата": Поле для выбора даты операции с кнопками "Сегодня", "Вчера". Обязательно для заполнения.
"Зачислить на счёт": Выпадающий список, содержащий все активные счета текущего профиля. Обязательно для заполнения.
"Категория": Выпадающий список категорий доходов (например, "Зарплата", "Фриланс", "Подарки"). Обязательно для заполнения.
"Подкатегория": Выпадающий список подкатегорий, связанных с основной категорией дохода.
"Сумма": Числовое поле для ввода суммы дохода. Обязательно для заполнения.
"Валюта": Выпадающий список валют. По умолчанию подставляется валюта выбранного счета.
"Примечание": Необязательное текстовое поле для комментариев.
Управляющие кнопки:
"ОК": Выполняет валидацию. При успехе сохраняет транзакцию, увеличивая сумму на балансе выбранного счета, и возвращает на экран списка доходов.
"ОТМЕНА": Закрывает форму без сохранения данных.
"ЕЩЕ": Сохраняет текущую транзакцию дохода и очищает форму для ввода следующей.
3.7.3. Экран "Фильтр"
Назначение: Предоставить пользователю возможность отобрать и отобразить в списке только те доходы, которые соответствуют заданным критериям.
Визуальное представление: Аналогично экрану фильтрации расходов.
Составные элементы:
Кнопка "СБРОСИТЬ ФИЛЬТР": Снимает все примененные фильтры.
Список критериев фильтрации:
"Счета": Список всех счетов с чекбоксами для выбора.
"Категории": Иерархический список категорий и подкатегорий доходов.
Логика работы фильтра: Идентична логике в модуле "Расходы" (ИЛИ внутри группы, И между группами).
Управляющие кнопки:
"ОК": Применяет фильтр и обновляет список доходов.
"ОТМЕНА": Закрывает экран без применения изменений
3.8. Модуль "Получено в кредит" (Учет задолженностей)
3.8.1. Экран списка кредитов
Назначение: Отображение списка всех полученных пользователем кредитов и займов. Отслеживание общей суммы долга и статуса погашения.
Способ вызова: Нажатие на иконку "Получено в кредит" на главном экране (дашборде).
Составные элементы экрана:
A. Верхняя панель (Header):
Элементы ("Бургер-меню", Заголовок "Получено в кредит", "Поиск" , "Фильтр" ) полностью аналогичны модулям "Расходы" и "Доходы".
B. Навигатор по периодам:
Аналогичен предыдущим модулям, позволяет фильтровать список по дате получения кредита.
C. Список кредитов:
Состояние "Список пуст": Отображается, если кредитов нет.
Состояние со списком: Отображает список кредитов. Каждый элемент списка должен содержать:
Имя кредитора.
Исходную сумму кредита.
Остаток долга (выделен цветом или отдельно).
Статус ("Не погашен" / "Погашен").
Поведение: Нажатие на запись в списке открывает ее для просмотра и редактирования в форме "Карточка кредитов".
D. Плавающая кнопка действия (FAB):
Описание: Круглая синяя кнопка с +.
Действие: При нажатии открывает экран "Карточка кредитов" для добавления новой записи о займе.
E. Итоговая панель:
Описание: Отображает три ключевых показателя по всем кредитам в списке (с учетом фильтров).
Строка 1 (черный цвет): Общая сумма полученных кредитов.
Строка 2 (черный цвет): Общая сумма возвращенных средств по этим кредитам.
Строка 3 (красный цвет): Общий остаток долга (Строка 1 - Строка 2). Красный цвет используется для акцентирования внимания на задолженности.
3.8.2. Форма добавления/редактирования кредита ("Карточка кредитов")
Назначение: Детальная форма для ввода и управления информацией о конкретном займе.
Визуальное представление: См. скриншот "16:19, Карточка кредитов".
Поля и элементы:
"Дата": Дата получения кредита. Стандартный выбор с "Сегодня", "Вчера".
"Имя кредитора": Выпадающий список, который подгружает данные из отдельного справочника "Кредиторы и должники". Должна быть возможность быстрого добавления нового кредитора из этой формы.
"Занести на счёт": Выпадающий список счетов пользователя. Важная логика: при сохранении кредита на баланс этого счета должна автоматически зачисляться сумма кредита.
"Сумма" и "Валюта": Сумма и валюта полученного займа.
"Процентная ставка, %" и "Вид ставки": Поля для информационного отслеживания условий кредита. Для MVP расчет процентов может не производиться автоматически. "Вид ставки" может быть списком ("единовременно", "в год", "в месяц").
"Сумма возврата": Поле, которое не редактируется вручную. Его значение автоматически рассчитывается на основе проведенных платежей по возврату.
"Статус": Выпадающий список ("Не погашен", "Погашен"). Статус должен автоматически меняться на "Погашен", когда "Сумма возврата" становится равной или больше "Суммы".
Ссылка "Возврат кредита": Ключевой элемент. При нажатии запускает сценарий погашения долга.
"Примечание": Текстовое поле для дополнительной информации.
Управляющие кнопки: "ОК" и "ОТМЕНА".
3.8.3. Экран "Фильтр"
Назначение: Отбор кредитов в списке по заданным критериям.
Критерии фильтрации:
"Счета": Позволяет отфильтровать кредиты по счету, на который они были зачислены.
"Кредиторы": Список всех кредиторов из справочника с чекбоксами для выбора.
"Статус": Тумблеры или чекбоксы для выбора кредитов со статусом "Не погашен" и/или "Погашен".
3.8.4. Бизнес-логика и сценарии
1. Создание кредита:
При сохранении новой "Карточки кредитов" система должна выполнить два действия:
Создать запись о кредите в модуле "Получено в кредит".
Автоматически создать доходную транзакцию на указанную "Сумму" на выбранный "Счёт", с категорией "Заёмные средства" или аналогичной. Это обеспечивает целостность баланса счетов.
2. Сценарий возврата кредита:
Пользователь открывает карточку существующего кредита и нажимает ссылку "Возврат кредита".
Открывается форма, похожая на "Карточку расходов", но специально для погашения долга. Поля: "Дата платежа", "Сумма платежа", "Списать со счёта" (откуда берутся деньги на погашение), "Примечание".
После сохранения платежа система:
Создает расходную транзакцию на "Сумму платежа" со счета погашения, с категорией "Погашение кредита".
Обновляет поле "Сумма возврата" в "Карточке кредитов", прибавляя к нему "Сумму платежа".
Проверяет, если "Сумма возврата" >= "Сумма", то автоматически меняет "Статус" кредита на "Погашен".

Состояние со списком: Отображает список созданных шаблонов. Каждый элемент должен содержать:
Название категории/примечание.
Сумму.
Периодичность (например, "Ежемесячно, 15-го числа", "Разово, 25.12.2024").
Дата следующего исполнения.
Поведение: Нажатие на запись открывает ее для редактирования в форме "Карточка планирования расходов".
C. Плавающая кнопка действия (FAB):
Круглая синяя кнопка с +.
Действие: При нажатии открывает "Карточку планирования расходов" для создания нового шаблона.
D. Итоговая панель:
Отображает общую сумму всех запланированных расходов, у которых следующая дата исполнения попадает в выбранный на экране период.
3.10.2. Форма планирования расхода ("Карточка планирования расходов")
Назначение: Форма для создания и редактирования шаблона будущего или регулярного расхода.
Ключевое отличие от "Карточки расходов": Эта форма не создает транзакцию, а только сохраняет шаблон для будущего использования.
Поля и элементы:
"Дата": Дата начала действия плана. Для разового платежа — это дата его исполнения. Для регулярного — дата первого платежа.
"Период": Ключевое поле. Выпадающий список с вариантами периодичности:
"Однажды" (разовый платеж).
"Каждый день".
"Каждую неделю".
"Каждый месяц" (по умолчанию).
"Каждый год".
Остальные поля ("Списать со счёта", "Категория", "Сумма" и т.д.) полностью идентичны полям в "Карточке расходов" (см. п. 3.6.2). Они определяют, какая транзакция будет создана в будущем.
Управляющие кнопки:
"ОК": Сохраняет шаблон (план) и возвращает на предыдущий экран.
"ОТМЕНА": Закрывает форму без сохранения.
"ЕЩЕ": Сохраняет текущий шаблон и очищает форму для создания следующего.
3.10.3. Экран "Фильтр"
Назначение: Фильтрация списка созданных планов по счетам и категориям.
Визуальное представление и функционал: Полностью идентичен экрану фильтрации в модуле "Расходы".
3.10.4. Бизнес-логика и сценарий исполнения плана
Это самая важная часть модуля, определяющая его пользу.
1. Создание плана: Пользователь создает и сохраняет "Карточку планирования". В базе данных создается запись о будущем/регулярном платеже, но баланс счетов не изменяется.
2. Автоматическое исполнение плана (основной сценарий):
На сервере должен работать ежедневный автоматический скрипт (планировщик задач/cron job).
Каждый день (например, в 01:00) этот скрипт проверяет все активные планы всех пользователей.
Если у плана дата следующего исполнения совпадает с текущей датой, скрипт:
Автоматически создает новую расходную транзакцию, используя все данные из шаблона (счет, категория, сумма и т.д.). Баланс соответствующего счета уменьшается.
Если план был "разовый" ("Однажды"), он помечается как "Исполнен" и становится неактивным.
Если план "регулярный", система вычисляет и записывает для него новую дату следующего исполнения (например, +1 месяц).
Система отправляет пользователю уведомление (push или email): "Запланированный расход '[Название категории]' на сумму [Сумма] был автоматически проведен."
3. Альтернативный сценарий (с подтверждением пользователя):
Как опция в настройках приложения. Вместо автоматического создания транзакции, система в день исполнения плана отправляет пользователю уведомление: "Сегодня запланирован расход '[Название категории]'. Подтвердить списание?". При нажатии на уведомление открывается предзаполненная "Карточка расхода" для подтверждения. Этот сценарий дает пользователю больше контроля.

3.12. Модуль "Бюджет расходов"
3.12.1. Экран списка бюджетов
Назначение: Отображение списка всех установленных лимитов (бюджетов) по категориям расходов на выбранный период и отслеживание прогресса их исполнения.
Способ вызова: Нажатие на иконку "Бюджет расходов" на главном экране (дашборде).
Составные элементы экрана:
A. Верхняя панель:
Иконка "Бургер-меню".
Заголовок: "Бюджет расходов".
Иконка "Поиск": Позволяет искать бюджет по названию категории.
Иконка "Копировать": Ключевая функция. При нажатии открывает экран "Скопировать бюджет".
B. Навигатор по периодам:
Аналогичен предыдущим модулям, позволяет выбрать месяц и год для отображения бюджетов.
При нажатии на название месяца ("Октябрь 2025") открывается модальное окно для быстрого выбора месяца и года.
C. Список бюджетов:
Состояние "Список пуст": Отображается, если на выбранный период не установлено ни одного бюджета.
Состояние со списком (ключевой интерфейс): Отображает список установленных бюджетов. Каждый элемент списка должен содержать:
Название категории (и подкатегории, если применимо).
Прогресс-бар: Визуальный индикатор исполнения бюджета. Цвет прогресс-бара должен меняться в зависимости от процента трат (например, до 75% - зеленый, 75-95% - желтый, свыше 95% - красный).
Цифровая информация: Отображение фактических трат и установленного лимита (например, "Потрачено 7 500 из 10 000 р").
Остаток: Сумма, оставшаяся до достижения лимита.
Поведение: Нажатие на запись в списке открывает ее для редактирования в форме "Карточка бюджета расходов".
D. Плавающая кнопка действия (FAB):
Круглая синяя кнопка с +.
Действие: При нажатии открывает "Карточку бюджета расходов" для создания нового лимита.
E. Итоговая панель:
Отображает два агрегированных показателя: [Сумма фактических расходов] ([Общая сумма лимитов]). Например, "15 200 р (50 000)". Это дает пользователю общее понимание исполнения бюджета за месяц.
3.12.2. Форма создания/редактирования бюджета ("Карточка бюджета расходов")
Назначение: Форма для установки или изменения лимита расходов для конкретной категории на определенный период.
Визуальное представление: См. скриншот "17:00, Карточка бюджета расходов".
Поля и элементы:
"Период": Выпадающий список для выбора месяца и года, на который устанавливается бюджет.
"Категория": Выпадающий список категорий расходов. Обязательно для заполнения.
"Подкатегория": Выпадающий список, зависимый от основной категории. Позволяет установить более гранулярный бюджет.
"Сумма": Числовое поле для ввода суммы лимита. Обязательно для заполнения.
"Валюта": Выпадающий список валют, доступных в профиле.
"Примечание": Необязательное текстовое поле.
Управляющие кнопки: "ОК" и "ОТМЕНА".
3.12.3. Функция "Скопировать бюджет"
Назначение: Предоставить пользователю быстрый способ перенести структуру бюджета с одного месяца на другой, чтобы не создавать все лимиты заново.
Визуальное представление: См. скриншот "17:00, Скопировать бюджет".
Поля и элементы:
"Из": Выпадающий список для выбора исходного периода (месяц и год), из которого будут скопированы данные.
"В": Выпадающий список для выбора целевого периода (месяц и год), в который будут вставлены данные.
Тумблер "Скопировать бюджет расходов": Включает/выключает копирование лимитов по расходам.
Тумблер "Скопировать бюджет доходов": [Заготовка] Предполагает наличие симметричного модуля "Бюджет доходов".
Логика работы:
Пользователь выбирает исходный и целевой месяцы.
Активирует нужные тумблеры.
Нажимает "ОК".
Система находит все записи о бюджетах за исходный месяц и создает их точные копии для целевого месяца.
Бизнес-правило: Если в целевом месяце уже существуют какие-либо бюджеты, они должны быть полностью заменены новыми (перезаписаны). Система должна выводить предупреждение: "Все существующие бюджеты на [целевой месяц] будут удалены. Продолжить?".
3.12.4. Бизнес-логика и сценарии
1. Связь с фактическими расходами:
Ключевая логика модуля заключается в постоянном сравнении установленных лимитов с фактическими тратами.
Для каждого элемента в списке бюджетов (например, "Продукты" на "Октябрь 2025") система должна в реальном времени рассчитывать сумму всех расходных транзакций, у которых категория="Продукты" и дата попадает в "Октябрь 2025".
2. Уведомления (важное нефункциональное требование):
Система должна отправлять пользователю push- или email-уведомления при достижении определенных порогов бюджета по любой из категорий:
При достижении 90% лимита (предупреждение).
При достижении 100% лимита (уведомление о превышении).
3.13. Модуль "Бюджет доходов"
3.13.1. Экран списка бюджетов по доходам
Назначение: Отображение списка всех запланированных сумм (бюджетов) по категориям доходов на выбранный период и отслеживание прогресса их получения.
Способ вызова: Нажатие на иконку "Бюджет доходов" на главном экране (дашборде).
Визуальное представление: Аналогично экрану "Бюджет расходов", с соответствующими изменениями в терминологии.

Иконка "Бургер-меню".

Иконка "Поиск" : Поиск бюджета по названию категории дохода.

Название категории дохода (например, "Зарплата", "Фриланс").

Остаток: Сумма, которую осталось получить для выполнения плана.
Поведение: Нажатие на запись в списке открывает ее для редактирования в форме "Карточка бюджета доходов" (см. п. 3.13.2).
D. Плавающая кнопка действия (FAB):
Круглая синяя кнопка с +.

E. Итоговая панель:
Отображает два агрегированных показателя: [Сумма фактических доходов] ([Общая сумма запланированных доходов]). Например, "115 000 р (120 000)".
3.13.2. Форма создания/редактирования бюджета ("Карточка бюджета доходов")
Назначение: Форма для установки или изменения плана поступлений для конкретной категории на определенный период.
Визуальное представление: Аналогично "Карточке бюджета расходов".
Поля и элементы:
"Период": Выпадающий список для выбора месяца и года.
"Категория": Выпадающий список категорий доходов. Обязательно для заполнения.
"Подкатегория": Выпадающий список подкатегорий доходов.
"Сумма": Числовое поле для ввода запланированной суммы дохода. Обязательно для заполнения.
"Валюта": Выпадающий список валют.
"Примечание": Необязательное текстовое поле.
Управляющие кнопки: "ОК" и "ОТМЕНА".
3.13.3. Бизнес-логика и сценарии

Для каждого элемента в списке бюджетов (например, "Зарплата" на "Октябрь 2025") система должна в реальном времени рассчитывать сумму всех доходных транзакций, у которых категория="Зарплата" и дата попадает в "Октябрь 2025".

Раздел 4. Модули управления справочниками
4.1. Модуль "Категории расходов"
Назначение: Управление списком категорий и подкатегорий, которые используются для классификации расходных операций.
Способ вызова: Нажатие на иконку "Категории расходов" на главном экране (дашборде).
Состояние по умолчанию: Приложение должно поставляться с предустановленным базовым набором категорий (Продукты, Транспорт, ЖКХ и т.д.).
4.1.1. Экран списка категорий
Визуальное представление: См. скриншот "17:05, Категории расходов".
Элементы:
Header с заголовком "Категории расходов".
Простой текстовый список всех категорий верхнего уровня.
Плавающая кнопка + для добавления новой категории.
Footer с информацией о количестве записей ("Строк: N").
Поведение:
Нажатие на +: Открывает форму добавления новой категории.
Нажатие на существующую категорию: Открывает экран со списком подкатегорий для этой категории. Этот экран должен выглядеть так же, но с заголовком "Подкатегории для '[Название категории]'" и позволять добавлять/редактировать/удалять уже подкатегории.
Длительное нажатие: Открывает контекстное меню ("Редактировать", "Удалить").
4.1.2. Форма добавления/редактирования категории
Визуальное представление: См. скриншот "17:05, Введите категорию расхода".
Элементы: Текстовое поле для ввода названия, кнопки "ОК" и "ОТМЕНА".
Валидация: Название категории должно быть обязательным для заполнения и уникальным на своем уровне вложенности.
4.1.3. Бизнес-логика:
Удаление: Категорию или подкатегорию нельзя удалить, если она используется хотя бы в одной транзакции (в модуле "Расходы") или в одном бюджете/плане. При попытке удаления система должна выводить ошибку: "Невозможно удалить категорию, так как она используется".
4.2. Модуль "Категории доходов"
Назначение: Управление списком категорий и подкатегорий для классификации доходных операций.
Функционал: Модуль полностью идентичен модулю "Категории расходов", но работает со справочником категорий для доходов (Зарплата, Подарки, Фриланс и т.д.). Все экраны и логика (включая работу с подкатегориями и правила удаления) аналогичны.
4.3. Модуль "Единицы измерения"
Назначение: Управление справочником единиц измерения, которые можно использовать при добавлении расходов (например, для указания количества товара).
Способ вызова: Нажатие на иконку "Единицы измерения" на главном экране.
Состояние по умолчанию: Рекомендуется предустановить базовый набор единиц (шт, кг, л, м, и т.д.).
4.3.1. Экран списка единиц измерения
Элементы и поведение: Аналогичны экрану списка категорий, но без иерархии (это плоский список). Длительное нажатие позволяет редактировать/удалять.
4.3.2. Форма добавления/редактирования
Элементы и валидация: Аналогичны форме для категорий.
4.3.3. Бизнес-логика:
Удаление: Единицу измерения нельзя удалить, если она используется хотя бы в одной транзакции.
4.4. Модуль "Кредиторы и должники"
Назначение: Управление единым справочником контактов, которые могут выступать в роли как кредиторов (у кого вы берете в долг), так и должников (кто берет в долг у вас).
Способ вызова: Нажатие на иконку "Кредиторы и должники" на главном экране.
4.4.1. Экран списка
Элементы:
Header с заголовком.
Панель вкладок: "КРЕДИТОРЫ" и "ДОЛЖНИКИ". Позволяет фильтровать общий список контактов.
Список контактов (изначально пуст).
Плавающая кнопка + для добавления нового контакта.
Поведение:
При нажатии на + открывается форма добавления нового контакта.
При нажатии на контакт в списке открывается форма для его редактирования.
4.4.2. Форма добавления/редактирования контакта
Элементы:
Текстовое поле для ввода имени/названия контакта.
[Рекомендация] Добавить чекбоксы: "Может быть кредитором", "Может быть должником". Это позволит контакту появляться в соответствующих выпадающих списках в модулях "Получено в кредит" и "Выдано в долг".
Кнопки "ОК" и "ОТМЕНА".
Валидация: Имя контакта должно быть обязательным и уникальным.
4.4.3. Бизнес-логика:
Удаление: Контакт нельзя удалить, если он связан хотя бы с одной записью в модулях "Получено в кредит" или "Выдано в долг".
Раздел 5. Модуль "Настройки"
Назначение: Предоставить пользователю централизованный интерфейс для управления всеми аспектами работы приложения, включая внешний вид, безопасность, резервное копирование и другие параметры.
Способ вызова: Нажатие на иконку "Настройки" (шестеренка) в главном меню или навигационной панели.
5.1. Секция "Внешний вид"
5.1.1. При входе показывать:
Описание: Позволяет выбрать, какой экран будет открываться по умолчанию при запуске приложения.
Тип: Выпадающий список.
Опции: "Главное окно навигации" (дашборд с иконками), "Последний открытый экран", "Форма добавления расхода".
5.1.2. Стиль интерфейса:
Описание: Позволяет переключать цветовую тему приложения.
Тип: Выпадающий список.
Опции: "Светлый", "Темный", "Системный" (автоматически подстраивается под настройки ОС).
5.1.3. Язык интерфейса:
Описание: Позволяет изменить язык приложения.
Тип: Выпадающий список.
Опции: "Системный" (по умолчанию), "Русский", "English". (Требует внедрения механизма локализации/i18n).
5.2. Секция "Настройки списков"
5.2.1. Размер шрифта:
Описание: Позволяет изменить базовый размер шрифта в приложении для улучшения читаемости.
Тип: Выпадающий список.
Опции: "Маленький", "Средний", "Большой".
5.3. Секция "Конфиденциальность"
5.3.1. Включить вход по пин-коду:
Описание: Добавляет дополнительный локальный уровень безопасности.
Тип: Тумблер (переключатель).
Логика: При первом включении тумблера система должна запустить сценарий создания PIN-кода: "Введите PIN-код" -> "Подтвердите PIN-код". PIN-код хранится локально на устройстве в зашифрованном виде.
5.3.2. Автоблокировка приложения:
Описание: Определяет время бездействия, после которого приложение заблокируется и потребует ввода PIN-кода (если он включен).
Тип: Выпадающий список. Доступен только если включен PIN-код.
Опции: "Немедленно", "Через 10 секунд", "Через 1 минуту", "Через 5 минут".
5.3.3. Диспетчер задач:
Описание: Повышает конфиденциальность, скрывая содержимое окна приложения в списке недавних приложений ОС и запрещая создание скриншотов.
Тип: Чекбокс. (Реализация зависит от платформы, в основном актуально для Android).
5.4. Секция "Резервное копирование и синхронизация"
5.4.1. Резервное копирование:
Описание: Автоматически создает локальную резервную копию базы данных.
Тип: Чекбокс "Создавать резервную копию базы данных при закрытии программы".
5.4.2. Количество резервных копий:
Описание: Ограничивает количество хранимых локальных копий для экономии места (ротация).
Тип: Числовое поле или выпадающий список.
Опции: "3", "5", "10".
5.4.3. Шифровать данные:
Описание: Включает шифрование для файлов резервных копий и экспортируемых отчетов.
Тип: Чекбокс.
5.4.4. Подключение облачных сервисов (Dropbox, Яндекс Диск):
Описание: Позволяет подключить облачное хранилище для автоматической выгрузки резервных копий.
Тип: Кнопки "Подключить...".
Логика:
При нажатии запускается процесс авторизации OAuth 2.0 в выбранном сервисе.
Приложение запрашивает доступ к своей выделенной папке.
После успешного подключения кнопка меняется на "Отключить [имя_аккаунта]".
При создании резервной копии она автоматически загружается в облако.
5.4.5. Настройки Wi-Fi (SMB):
Описание: [Продвинутый функционал] Позволяет настроить резервное копирование на сетевой диск (NAS) в локальной сети.
Тип: Кнопка, открывающая форму с полями: "Адрес сервера", "Имя папки", "Пользователь", "Пароль".
5.5. Секция "Другие настройки"
5.5.1. Распознавать СМС:
Описание: [Сложный функционал] Автоматически создает транзакции, анализируя СМС-сообщения от банков.
Тип: Чекбокс.
Логика:
При включении приложение запрашивает у пользователя разрешение на чтение СМС.
В приложении должен быть раздел для настройки шаблонов парсинга для разных банков (например, "Сбербанк: 'Покупка N руб. Карта *1234. Баланс...'").
При поступлении нового СМС система пытается применить к нему шаблон, и в случае успеха создает черновик транзакции, который пользователь должен подтвердить.
5.5.2. Получать уведомления:
Описание: Главный переключатель для всех push-уведомлений приложения (о бюджетах, плановых операциях и т.д.).
Тип: Чекбокс.
5.5.3. Расположение рабочей папки:
Описание: Информационное поле, которое показывает путь к локальным файлам приложения. Не редактируется.
5.5.4. Анонимная статистика:
Описание: Позволяет пользователю разрешить или запретить отправку анонимных данных об использовании приложения для улучшения продукта.
Тип: Чекбокс.
Раздел 6. Дополнительные и служебные модули
6.1. Монетизация ("Премиум версия" и "Активация")
Назначение: Предоставить пользователям возможность приобрести расширенную версию приложения и активировать ее.
6.1.1. Экран "Премиум версия"
Способ вызова: Нажатие на пункт "Премиум версия" в главном меню.
Предполагаемый функционал: Открывается информационный экран (лендинг), который описывает преимущества платной версии, например:
Неограниченное количество профилей.
Расширенные отчеты и экспорт в PDF.
Автоматическая синхронизация с облаком.
Отсутствие рекламы.
На экране должны быть четко указаны стоимость и условия подписки (например, "999 р./год" или "2999 р. навсегда") и кнопка "Купить" / "Подписаться", которая инициирует процесс оплаты через App Store / Google Play или платежный шлюз для веб-версии.
6.1.2. Экран "Активация"
Способ вызова: Нажатие на пункт "Активация" в главном меню.
Предполагаемый функционал: Открывается простой экран с одним полем ввода "Введите ключ активации" и кнопкой "Применить". Предназначен для активации премиум-версии с помощью промокодов или ключей, полученных не через стандартную покупку в сторе.
6.2. Управление базой данных
Назначение: Предоставить пользователю инструменты для обслуживания локальной базы данных.
6.2.1. Функция "Очистка базы данных"
Предполагаемый функционал: При выборе этого пункта система должна показать критически важное предупреждение: "Вы уверены, что хотите полностью очистить базу данных? Все ваши транзакции, счета, бюджеты и настройки будут удалены безвозвратно. Это действие нельзя отменить." Для подтверждения пользователь должен вручную ввести слово "УДАЛИТЬ".
6.2.2. Функция "Сжать базу данных"
Предполагаемый функционал: Запускает техническую процедуру оптимизации файла базы данных (аналог VACUUM в SQLite) для уменьшения его размера. Перед началом должен выводиться простой диалог: "Начать оптимизацию базы данных? Это может занять некоторое время."
6.3. Информационные разделы
6.3.1. "Справка": Открывает экран с часто задаваемыми вопросами (FAQ) или ссылку на веб-страницу с документацией по использованию приложения. Обязательно должен содержать способ связи со службой поддержки (email или форма обратной связи).
6.3.2. "О программе": Открывает стандартный экран "About" с указанием названия приложения, текущей версии, года и правообладателя, а также ссылками на "Политику конфиденциальности" и "Пользовательское соглашение".
6.3.3. "Поделиться с друзьями": Инициирует вызов стандартного системного окна "Поделиться". В качестве контента передается заранее заготовленный текст (например, "Я веду учет финансов в приложении FinanceTracker! Попробуй и ты: [ссылка на приложение]").
6.4. Журналы импорта данных
Назначение: Предоставить пользователю возможность просмотреть исходные данные, на основе которых были созданы автоматические транзакции.
6.4.1. "Журнал СМС": Открывает экран со списком СМС, которые были распознаны приложением как финансовые. Для каждой записи указывается текст СМС, дата и статус ("Транзакция создана", "Пропущено").
6.4.2. "Список чеков": Аналогично журналу СМС, но для отсканированных QR-кодов. Отображает список чеков с датой, распознанной суммой и статусом обработки.
6.5. Системные функции
6.5.1. Уведомление о новой версии
Визуальное представление: Зеленая плашка внизу экрана с текстом "Доступна новая версия" и кнопками "ОБНОВИТЬ" и "СКРЫТЬ".
Логика: Приложение при запуске должно обращаться к серверу для проверки последней доступной версии. Если текущая версия устарела, показывается эта плашка.
Действия:
"ОБНОВИТЬ": Перенаправляет пользователя в App Store / Google Play или на страницу загрузки веб-приложения.
"СКРЫТЬ": Закрывает плашку до выхода следующей новой версии.
